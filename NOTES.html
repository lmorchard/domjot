General Notions

I've got some general thoughts on how this thing should be built:

  • Documents are a microformat pattern of HTML5 - eg. article containing note sections
  • Authoring UI should be considered scaffolding atop documents, easily removable and not necessary for viewing
  • Navigation augmentations (eg. search, hide/show links, sidebar listing pages) should also be optional scaffolding
  • Play with media queries and making this thing work great on my Android phone
  • Plugins driven by Backbone events
  • The code base should eventually do interesting things in web pages, browser add-ons, and server-side apps.

Markdown editor

I want to build a plugin that lets me write notes with Markdown

However, I can't find an easy way to do round-trip edits between HTML and Markdown with JS. So, I might just stash Markdown source in a hidden PRE element and render to HTML on save. Then, I'll load the editor from the PRE on the next edit. The main downside will be that any edits directly done to the HTML will be clobbered. Probably covered by a caveat warning.

Might be a good idea to advise plugins to store extended content outside the body content div, so that editors like this don't clobber their content.
